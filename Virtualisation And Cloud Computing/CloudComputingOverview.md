# Cloud Computing

Cloud computing provides an effective solution to challenges such as moving your application from a single laptop to an online environment that can be accessed from anywhere.

Instead of running an application on a single computer in one location, cloud computing allows you to use computing resources over the internet. This approach improves accessibility, reliability, and scalability. As more users begin to access the application, cloud infrastructure can adapt to meet increased demand.


## How Servers Evolved to the Cloud

Cloud computing did not emerge suddenly. It is the result of decades of evolution in how servers are deployed, managed, and optimized.

Over time, organizations sought to:

- Reduce operational costs
- Improve hardware utilization
- Simplify infrastructure management
- Increase scalability and availability

Each stage in this evolution improved efficiency and flexibility, eventually leading to modern cloud computing models. The progression typically moves from physical servers to virtualization, and ultimately to cloud-based infrastructure.

## Cloud Benefits and Characteristics

As applications evolved from physical servers to cloud infrastructure, it became clear why cloud computing is widely adopted today. The cloud was designed to address common challenges such as limited capacity, high infrastructure costs, and slow scalability.

The following characteristics explain how cloud computing simplifies application deployment, scaling, and management:

- **Scalability** — Resources can be scaled up or down as application demand changes.
- **On-demand self-service** — Servers, storage, and networking resources can be provisioned instantly without waiting for physical hardware.
- **Pay-as-you-go pricing** — Costs are based on actual resource usage rather than large upfront investments.
- **Security** — Cloud providers implement strong infrastructure-level security controls.
- **High availability** — Applications remain operational even if individual components fail.
- **Global accessibility** — Applications can be accessed from anywhere with an internet connection.

In summary, cloud computing enables IT resources to be flexible, cost-efficient, and easier to manage.


## Types of Cloud (Deployment Models)

Cloud computing offers different deployment models depending on organizational needs, control requirements, and compliance considerations.

### Public Cloud
Used by startups, web platforms, and globally distributed applications. It is cost-effective, highly scalable, and requires no direct infrastructure management.

### Private Cloud
Used by banks, healthcare institutions, and government organizations. It offers greater control, customization, and regulatory compliance for sensitive workloads.

### Hybrid Cloud
Combines public and private environments. Organizations may keep sensitive data in a private cloud while leveraging public cloud resources to handle high demand or scalability needs.


## Cloud Service Models

Cloud services are also categorized based on the level of responsibility shared between the provider and the customer.

### Infrastructure as a Service (IaaS)
You rent fundamental computing resources such as virtual machines, storage, and networking.  
You manage the operating system and applications, while the provider manages the physical infrastructure.

### Platform as a Service (PaaS)
The provider manages the infrastructure and operating system.  
You focus on building, deploying, and maintaining your application without managing servers directly.

### Software as a Service (SaaS)
You use a fully managed application over the internet.  
The provider manages the entire infrastructure and application stack. Examples include Gmail and Zoom.
